Introduction

Grauman is a framework-agnostic HTML5 File viewer that can view MP4, M4A, Encrypted HLS M3U8 Playlist, PDF, JPEG, GIF, PNG, and BMP files natively in a browser.

MediaFile

Construct a new MediaFile, which is an object that tells the MediaPlayer how to display a file at a given URL.

The MediaPlayer is optimized to work with files that have been processed by the Wiredrive asset ingestion pipeline (wd-thumb); specifially, video and audio assets that have been processed by selective-transcode (transmux.py). For more information, see #ingest-pipeline.

new MediaFile(file: Object): MediaFile
Parameters
file (Object) A hash to initialize the MediaFile with.
Name Description
file.extension string Extension of the file.
file.mimeType string MIME Type of the file.
file.url string URL of the file to display.
file.duration number? Total playback length, in seconds.
file.poster string? (default null) URL to an image to use as a poster image.
file.is360 boolean? (default false) Is the file content formatted for a 360/panoramic viewer?
file.stereoscopicLayout STEREOSCOPIC_LAYOUTS? (default NONE) The stereoscopic layout of the content, if applicable.
file.height number? (default 0) The intended display height, in pixels, of the file.
file.width number? (default 0) The intended display width, in pixels, of the file.
file.fps number? (default 0.0) Frames-per-second playback speed of video content.
file.channels number? (default 0) Number of audio channels.
Returns
MediaFile: The newly instantiated MediaFile object.
Throws
  • TypeError: Thrown if a required property on the file object hash is of the wrong type.

MediaPlayer

Constructs a new MediaPlayer, which is a web component for viewing media assets in a Web Browser.

new MediaPlayer(container: HTMLElement, settings: Object??): MediaPlayer
Parameters
container (HTMLElement) An HTMLElement DOM node that will be the MediaPlayer instance's main view element. The MediaPlayer uses this element to determine it's maximum allowed size.

See #container-sizing for more information on how the MediaPlayer uses the container to calculate its size

settings (Object?? = {}) A hash of config values to initialize the MediaPlayer with.
Name Description
settings.autoplay boolean Start playing the MediaFile automatically after it loads.
settings.file MediaFile A MediaFile to load immediately after initializing
settings.keyboardShortcutsEnabled boolean A flag to enable or disable this MediaPlayer's keyboard shortcut bindings.
settings.localStorageEnabled boolean A flag to enable or disable this MediaPlayer reading from and writing to browser localStorage.
settings.localStorageKey string The key that this MediaPlayer instance will use to read from and write to localStorage, if it is enabled.
settings.loop boolean When the Media stream has reached the end, automatically start playing the media from the beginning.
settings.playbackSpeed number TODO
settings.volume number TODO
Returns
MediaPlayer: The newly instantiated MediaPlayer object
Throws
  • TypeError: If the container parameter is not supplied or is not an instance of an HTMLElement
Example
var container = document.querySelector('#your-container');
var mediaPlayer = new MediaPlayer(container);

// or, with a settings object
// TODO: make the indentation not suck when documentation renders it
var mediaPlayer = new MediaPlayer(container, {
autoplay: true,
volume: 0.5,
localStorageEnabled: false,
file: new MediaFile({
extension: 'mp4',
mimeType: 'video/mp4',
url: 'http://example.com/clip.mp4'
})
});
Static Members
▸ publicPath

Set's the URL path to the MediaPlayer's other build chunks.

Since MediaPlayer uses Webpack's code splitting feature, which caches the chunks after they are dynamically requested, it is highly recommended that you not modify this property after the first MediaPlayer instance is created.

Furthermore, since many aspects of JavaScript are subject to CORS restrictions, it is highly recommended that you keep the chunks located on the same origin as the main media-player index file.

See #code-splitting for more information.

publicPath
Instance Members
▸ stop()

Stops playback. Equivalent to pausing playback and seeking back to the beginning. Has no effect if playback is already stopped.

stop()
▸ seek(time)

Seek/jump forwards or backwards by the specified relative time. See seekTo.

If the relative time given will cause the file to seek past its duration, it will seek to the end of the file. If the relative time given causes the file to seek before its beginning, it will seek to the beginning of the file.

Seeking does not change playback state.

seek(time: number)
Parameters
time (number) The relative time, in seconds, to seek by. Postive numbers seek forwards in time, negative numbers seek backwards in time.
▸ seekTo(time)

Seek/jump to the specified time. See seek.

If the given time is greater than the file's playback duration, the player will seek to the end of the file.

Does not change the playback state, i.e. if the MediaPlayer is playing, it will seek to the specified time and resume playing from that time. If the player is paused, it will seek to the specified time and remain paused.

seekTo(time: number)
Parameters
time (number) The playback time, in seconds, to seek to.
▸ play()

Tells the MediaPlayer to play the file, starting from the beginning if the file has not yet been played. If the player is paused, playback will resume from the pause point. If playback has ended, it will restart from the beginning.

Calling play while the MediaPlayer is already playing has no effect.

play()
▸ pause()

Tells the MediaPlayer to pause playback. Has no effect if the MediaPlayer is paused.

A read-only boolean indicating whether or not the environment supports programmatically entering and exiting fullscreen mode. If this property is false, the UI fullscreen button will not be visible and any attempt to change this instances fullscreen property will throw an Error

▸ isVolumeControllable

A read-only boolean indicating whether or not the environment supports programatically changing the volume of a MediaElement (iOS does not currently support this). If this property is false, the UI will not show the volume slider (you can still toggle mute) and setting the volume property on this instance will fail silently.

▸ timeupdate

timeupdate event. Fired to indicate that the currentTime of playback has changed.

User agents vary their frequency of this event depending on system load. The average frequency of this event being emitted could be between 4 and 70 times per second. If you are listening to this event and triggering a UI redraw or any kind of heavy update in direct response to your event handler for this event, it is highly recommended you throttle or debounce your redraws.

Better yet, just throttle every callback you bind to this event. You've been warned.

Binding to this event is the same as binding directly to the timeupdate event on an HTMLMediaElement object.

UPSCALE_MODES

Enume for the various modes of upscaling the MediaPlayer can be set to. Upscaling is the process of resizing the MediaPlayer beyond the dimensions of the given MediaFile to completely fill the area of the MediaPlayer's container without skewing the MediaFile's aspect ratio.

UPSCALE_MODES
Static Members
▸ FULLSCREEN_ONLY

Content will only be upscaled when the MediaPlayer is in fullscreen mode.

FULLSCREEN_ONLY
▸ ALWAYS

Content will always be upscaled

ALWAYS
▸ NEVER

Content will never be upscaled. The MediaPlayer will never scale beyond the dimensions of the MediaFile, even when the MediaPlayer is in fullscreen mode.

NEVER
